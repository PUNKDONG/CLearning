# Day11：只学 `realloc`

## 1. 今天的目标
今天只做一件事：把 `realloc` 真正学会。

学会的标准：
1. 你知道 `realloc` 成功和失败各会发生什么。
2. 你能写出“失败不丢数据”的安全写法。
3. 你能解释为什么不能直接 `p = realloc(p, ...)`。

---

## 2. `realloc` 是什么
`realloc` 用来“调整已分配内存的大小”。

函数原型：
```c
void *realloc(void *ptr, size_t new_size);
```

你可以把它理解成：
- 原来有一块内存（`ptr`）。
- 现在想把它变大或变小（`new_size`）。

---

## 3. `realloc` 的三种结果（核心）
调用：
```c
new_ptr = realloc(ptr, new_size);
```
只会出现三种情况：

1. 成功，地址不变
- `new_ptr == ptr`
- 说明原地调整成功

2. 成功，地址改变
- `new_ptr != ptr`
- 说明系统搬家了，旧地址不能再用

3. 失败，返回 `NULL`
- 重点：**旧指针 `ptr` 仍然有效**
- 这就是为什么不能直接覆盖旧指针

---

## 4. 正确写法 vs 错误写法
错误写法：
```c
p = realloc(p, new_bytes);
if (p == NULL) {
    // 这里已经丢失旧地址
}
```

正确写法：
```c
int *tmp = realloc(p, new_bytes);
if (tmp == NULL) {
    // 失败：p 还有效，数据还在
    return -1;
}
p = tmp;
```

一句话：
- 先接到 `tmp`
- 成功后再覆盖 `p`

---

## 5. 两个特殊情况
1. `realloc(NULL, n)`
- 等价于 `malloc(n)`
- 可用于“从空指针开始分配”

2. `realloc(p, 0)`
- 各平台细节可能不同
- 初学阶段别用这个行为做核心逻辑
- 需要释放就直接 `free(p)` 更清楚

---

## 6. 为什么会崩：大小计算溢出
常见错误不是 `realloc` 本身，而是前面的计算溢出。

例如：
```c
new_cap = cap * 2;
bytes = new_cap * sizeof(int);
```
当 `cap` 很大，乘法可能溢出，结果变成很小值，最终导致越界写。

所以要做检查：
- 扩容前：`cap > SIZE_MAX / 2` -> 失败
- 算字节前：`new_cap > SIZE_MAX / sizeof(int)` -> 失败

---

## 7. 今天练习怎么做（只围绕 realloc）
- 练习1：写安全扩容函数（只练 `realloc` 失败语义）
- 练习2：在 `push` 里调用扩容（把 `realloc` 嵌入业务）
- 练习3：加上缩容（理解扩容和缩容是同一类问题）

你如果卡住，不要一次写完整题，按这顺序拆：
1. 先把“临时指针写法”写对
2. 再加返回值
3. 最后加边界检查

---

## 8. 今天结束前自检
你只需要检查这 5 条：
1. 有没有任何地方直接 `p = realloc(p, ...)`。
2. 失败时旧数据是否仍可访问。
3. 成功后是否更新了新指针。
4. 是否做了基本溢出检查。
