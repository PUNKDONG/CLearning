# 题目 2：TCP 文本统计服务器 + 客户端

目标：掌握 `socket`、`bind`、`listen`、`accept`、`connect`、`send`、`recv`，理解面向连接和字节流。

你需要写两个程序：
- `tcp_server.c`
- `tcp_client.c`

## 功能要求

服务端：
- 监听 `127.0.0.1:9002`
- 一次只处理一个客户端连接
- 客户端发送多行文本，以 `END\n` 作为结束标记
- 服务端在收到结束标记后，返回统计结果：
  - `LINES:<行数>`
  - `BYTES:<总字节数>`
  - `LONGEST:<最长一行长度>`
- 当前连接处理完成后，继续等待下一个客户端

客户端：
- 连接服务端
- 发送多行文本
- 最后一行发送 `END\n`
- 接收服务端统计结果并打印
- 打印后退出

## 约束

- 不能假设一次 `recv` 就能收完整消息
- 必须自己按换行符拆分数据
- 最多统计前 `1000` 行
- 如果超过 `1000` 行，返回 `ERR too many lines`
- 所有系统调用必须做错误检查

## 你应该重点掌握

- `listen` 和 `accept` 的职责区别
- 为什么 `accept` 后要用新的 fd 通信
- TCP 是字节流，不是消息边界天然分明的协议
- 循环 `recv` 与缓冲区拼接
- 客户端和服务端的关闭时机

## 自测清单

- 发送 3 行普通文本
- 包含空行
- 把一行拆成多次 `send`
- 构造超过 `1000` 行的输入

## 验收标准

- 服务端能稳定处理一个接一个的客户端
- 统计结果正确
- 拆包/粘包场景下不出错
