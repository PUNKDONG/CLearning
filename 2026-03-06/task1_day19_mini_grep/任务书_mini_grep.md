# mini_grep 任务书（训练模式）

## 1. 今日目标
通过实现一个简化版 `grep`，系统复习：
- C 语言基础语法与函数拆分
- 命令行参数解析（`argc/argv`）
- 文件读取与字符串匹配
- 错误处理与错误码设计
- 日志模块
- 多文件工程组织
- CMake 构建与 gdb 调试

## 2. 练习规则（强制）
1. 不抄完整答案。
2. 每次只做一个小步骤。
3. 你先写，我再检查。
4. 先提示、后修正，不直接重写你的代码。

## 3. 项目范围
工具名：`mini_grep`

功能要求：
1. 关键字检索
2. `-i` 忽略大小写
3. `-n` 输出行号
4. 参数校验
5. 错误处理
6. 日志输出

命令格式：
```bash
mini_grep [-i] [-n] <keyword> <file>
```

## 4. 第一阶段任务清单（按顺序）
### 步骤1：项目拆分（不写代码）
你需要先给出：
1. `main` 做什么
2. 参数解析模块做什么
3. 文件读取与匹配模块做什么
4. 日志模块做什么
5. 错误处理如何组织
6. 头文件和源文件如何划分

验收标准：
- 每个模块有明确边界（“做什么/不做什么”）
- 有可执行的文件划分方案（`.h/.c`）

### 步骤2：只实现参数解析模块
目标：先写 `parse_args()`，支持 `-i`、`-n`、非法参数报错。

### 步骤3：实现匹配函数
目标：实现大小写敏感与不敏感匹配。

### 步骤4：实现文件读取与输出
目标：逐行读文件，匹配后输出；`-n` 时输出行号。

### 步骤5：接入日志与统一错误码
目标：让程序能在失败路径可定位、可调试。

### 步骤6：接入 CMake + gdb 自测
目标：Debug 构建、断点调试关键路径。
