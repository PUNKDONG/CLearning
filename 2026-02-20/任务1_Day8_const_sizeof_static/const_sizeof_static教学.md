# Day8 教学：`const`、`sizeof`、`static`

这三个关键词非常高频，尤其是后续做嵌入式（驱动、协议栈、RTOS）时会天天见。

## 1. `const`：只读约束（编译期约束）

`const` 的核心作用：**告诉编译器“这个值不允许通过当前名字被修改”**。

### 1.1 最基础用法

```c
const int a = 10;
```

- 含义：`a` 是只读整数。
- 结果：写 `a = 20;` 会编译报错。

### 1.2 `const` 和指针（重中之重）

```c
const int *p1;      // 指向“只读 int”的指针：不能通过 p1 改值
int *const p2 = &x; // “指针本身只读”：p2 不能改指向
const int *const p3 = &x; // 值也不能改，指向也不能改
```

快速记忆：
- 看 `const` 在 `*` 左边还是右边。
- `const` 修饰谁，谁就“只读”。

## 2. `sizeof`：对象大小（单位是字节）

### 2.1 常见写法

```c
sizeof(int)
sizeof x
sizeof(arr)
sizeof(arr[0])
```

- 返回类型是 `size_t`（无符号整型）。
- 打印 `size_t` 用 `%zu`。

### 2.2 数组长度标准写法

```c
size_t n = sizeof(arr) / sizeof(arr[0]);
```

### 2.3 一个高频坑：函数参数里的数组

```c
void f(int arr[]) {
    sizeof(arr); // 这里不是整个数组大小，而是指针大小
}
```

原因：函数参数里的 `arr[]` 会退化为 `int *arr`。

## 3. `static`：把三个概念分开看（作用域 / 链接 / 存储期）

你之前卡住的点是正常的，因为很多资料把这三个词混在一起说。  
按 C 标准（6.2.2, 6.2.4）应分开理解：

1. 作用域（scope）：这个名字在代码哪一段可见。
2. 链接（linkage）：同名标识符能否跨文件指向同一个实体。
3. 存储期（storage duration）：对象在内存里活多久。

### 3.1 先记结论：`static` 在不同位置，语义不同

1. 在**块作用域**（函数体里）修饰变量：
   - 作用域：块内可见
   - 链接：无链接（不能跨文件通过同名访问）
   - 存储期：静态存储期（程序开始到结束都存在）

2. 在**文件作用域**修饰变量或函数：
   - 作用域：从声明处到本文件末尾
   - 链接：内部链接（只在当前翻译单元 `.c` 内可见）
   - 存储期（变量）：静态存储期

### 3.2 两段最常用代码

```c
void hit(void) {
    static int cnt = 0; // 块作用域 + 静态存储期 + 无链接
    cnt++;
    printf("%d\n", cnt);
}
```

- `cnt` 每次进函数不会重新创建。
- 只初始化一次，后续调用保留上次值。

```c
static int g_state = 0;      // 文件作用域 + 内部链接 + 静态存储期
static void helper(void) { } // 文件作用域函数 + 内部链接
```

- 其他 `.c` 文件不能 `extern` 到这两个名字。
- 这是 C 模块封装的核心手段（把实现细节藏在本文件）。

### 3.3 容易混淆的点

1. `static` 局部变量是“局部可见 + 全程存活”，不是“全局变量”。
2. “内部链接”只和名字可见性有关，不等于“生命周期短”。
3. 未显式初始化的静态存储期对象，会被默认初始化为 0。

### 3.4 一张判定表（写题时直接套）

| 声明位置 | 写不写 `static` | 作用域 | 链接 | 存储期 |
|---|---|---|---|---|
| 函数内 `int x;` | 不写 | 块作用域 | 无链接 | 自动存储期 |
| 函数内 `static int x;` | 写 | 块作用域 | 无链接 | 静态存储期 |
| 文件内 `int g;` | 不写 | 文件作用域 | 外部链接（默认） | 静态存储期 |
| 文件内 `static int g;` | 写 | 文件作用域 | 内部链接 | 静态存储期 |
| 文件内 `void f(void);` | 不写 | 文件作用域 | 外部链接（默认） | - |
| 文件内 `static void f(void);` | 写 | 文件作用域 | 内部链接 | - |

## 4. 三者在工程里的典型组合

1. 函数参数写成 `const uint8_t *buf`：表示函数只读输入缓冲区。
2. 用 `sizeof(buf)`/`sizeof(buf[0])` 计算静态数组长度。
3. 模块内部状态用 `static` 保存（例如通信状态机计数器）。

## 5. 你今天写题时的检查清单

1. `const` 是否写对位置（`const int *p` vs `int *const p`）。
2. `sizeof` 的输出是否使用 `%zu`。
3. 是否区分“main 里数组本体”和“函数参数里退化成指针”。
4. `static` 局部变量是否验证了“多次调用会保留值”。

## 6. 编译命令（建议固定）

```bash
gcc -Wall -Wextra -std=c11 文件名.c -o run
./run
```

- `-Wall -Wextra`：尽量提前暴露潜在 bug。
- `-std=c11`：使用 C11 标准，避免旧标准差异。
