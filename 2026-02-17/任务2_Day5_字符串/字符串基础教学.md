# Day5 字符串基础教学（C 语言）

## 1. C 字符串到底是什么

在 C 里，字符串本质是一个 `char` 数组，并且以 `'\0'` 结尾。

```c
char s[] = "abc";
```

内存里实际是：

`'a' 'b' 'c' '\0'`

`'\0'` 是结束标记，不是字符 `'0'`。

## 2. 常见定义方式

```c
char s1[] = "hello";          // 推荐，自动带 '\0'
char s2[6] = "hello";         // 手动给长度，刚好放下 hello + '\0'
char s3[] = {'h','i','\0'};   // 也可以手写
```

注意：如果忘了 `'\0'`，很多字符串函数会越界读。

## 3. 基本输入输出

```c
#include <stdio.h>

int main(void) {
    char s[100] = "hello";
    printf("%s\n", s);   // %s 打印字符串
    printf("%c\n", s[0]); // %c 打印单个字符
    return 0;
}
```

## 4. 手写 strlen 的核心思路

`strlen` 做的事：从头开始数，遇到 `'\0'` 停止。

```c
#include <stddef.h>

size_t my_strlen(const char *s) {
    size_t n = 0;
    while (s[n] != '\0') {
        n++;
    }
    return n;
}
```

为什么返回 `size_t`：
- 长度本来就是“非负大小”，标准库也是这个类型。

## 5. 为什么需要“安全拷贝”

危险写法（可能越界）：

```c
strcpy(dst, src);
```

如果 `dst` 太小，就会写爆内存。

安全拷贝函数要保证两件事：
1. 绝不写出 `dst` 边界。
2. 只要 `dst_size > 0`，结尾一定是 `'\0'`。

你今天的 `safe_strcpy` 就是在练这个。

## 6. 数组名 vs 指针（字符串场景）

```c
char s[] = "abc";
```

- `s` 是数组本体（定义处）
- 传参时会退化成 `char *`
- 所以函数里通常写 `const char *s` 或 `char *dst`

## 8. 高频坑位（务必避免）

1. 把 `'\0'` 和 `'0'` 混淆。
2. 用 `%d` 打印 `size_t`（应该用 `%zu`）。
3. 在函数里对参数数组用 `sizeof` 求长度（会退化成指针）。
4. 目标缓冲区不足时还硬拷贝。
5. 没检查输入函数返回值。
